Emergency Triage Assistant

Designed for high-pressure environments, this system combines intelligent retrieval, noise reduction, and predictive models to support faster emergency decisions.

Overview

Phase 1: What It Does

The Real-Time Emergency Response Triage Assistant is a system designed to quickly assess emergency situations using patient history and current symptoms. It retrieves relevant medical data, filters out unnecessary information, and provides a clear recommendation on what to do next.

Instead of requiring medical staff to go through large records or protocols, the system delivers focused, actionable insights within seconds, helping prioritize care in critical situations.

Phase 2: How It Achieves This

The system combines multiple techniques including data retrieval (RAG), machine learning classification, and contextual reasoning to produce accurate triage outcomes.

It analyzes patient history alongside current symptoms, removes irrelevant data through intelligent pruning, predicts possible conditions using a trained model, and then generates structured recommendations with explanations. This layered approach ensures both speed and reliability, even when working with large and noisy datasets.

Why use Triage Assistant?

Built to cut through noisy medical data and deliver precise emergency insights in real time.

✂️ Intelligent Context Pruning (Noise Reduction)

Irrelevant or outdated medical data is automatically removed to focus only on information related to the current condition. This reduces noise, improves accuracy, and significantly speeds up processing.

🧠 Machine Learning–Based Condition Classification

A trained Support Vector Machine (SVM) model analyzes symptoms along with patient history to predict possible medical conditions. This allows for fast and reliable initial classification.

🔗 Context-Aware Decision Support

The system combines current symptoms with filtered patient history to produce more accurate results. It takes into account past diagnoses, medications, and risk factors instead of relying on symptoms alone.

⚡ Latency-Optimized Processing Pipeline

The entire pipeline is designed for speed, using efficient retrieval and pruning techniques to deliver results in real time while tracking performance at each step.

How it works

1

Input

Enter patient ID and describe symptoms.

2

Patient History Retrieval

The system retrieves the patient's medical history for context.

3

Noise Reduction & Context Extraction

Extracts only: Relevant diagnoses, Medications, Risk factors

4

Disease Classification (SVM)

Symptoms + filtered history → fed into SVM model. Predicts possible condition.

5

LLM-Based Reasoning

Top condition + context → sent to LLM. LLM generates: Explanation

6

Triage Generation

Based on the predicted condition and context, the system generates a triage recommendation.
